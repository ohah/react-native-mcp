# 쿡북

MCP 도구를 조합하여 일반적인 개발 및 테스트 작업을 수행하는 실전 시나리오 모음입니다.

각 레시피는 초기 조사부터 검증까지 완전한 워크플로를 안내하며, AI 어시스턴트에 직접 붙여넣을 수 있는 대화형 프롬프트를 사용합니다.

## 시나리오

| 레시피                                 | 배울 수 있는 내용                                                 |
| -------------------------------------- | ----------------------------------------------------------------- |
| [UI 디버깅](./debug-ui)                | 컴포넌트 트리 검사, 요소 찾기, 상태 확인을 통한 UI 문제 진단      |
| [네트워크 모킹](./network-mocking)     | API 응답 모킹, 오류 및 느린 네트워크 시뮬레이션, 에러 핸들링 검증 |
| [성능 프로파일링](./performance)       | 렌더 프로파일링, 불필요한 리렌더 식별, 최적화 대상 탐색           |
| [비주얼 리그레션](./visual-regression) | 베이스라인 생성, 시각적 변경 감지, 요소 단위 스크린샷 비교        |
| [접근성 감사](./accessibility)         | 감사 실행, 위반 항목 식별, 수정 사항 검증                         |

## 레시피 사용 방법

이 레시피들은 도구 호출 시퀀스와 예제 응답으로 구성되어 있습니다. 실제로는 자연어로 원하는 작업을 설명하기만 하면 됩니다:

> "현재 화면의 스냅샷을 찍고 로그인 버튼을 찾아줘"

> "/api/users 엔드포인트가 에러를 반환하도록 모킹한 다음, 어떻게 되는지 확인해줘"

> "상품 목록을 스크롤하는 동안 렌더를 프로파일링해줘"

AI가 요청을 자동으로 적절한 도구 호출로 변환합니다.
